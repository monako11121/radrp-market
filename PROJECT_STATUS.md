## Что реализовано

### Авторизация
- Регистрация (username, email, пароль минимум 8 символов)
- Вход через NextAuth.js (JWT-стратегия)
- Дублирующиеся email/username — понятные ошибки
- Защита всех приватных маршрутов через `getServerSession`

### Объявления
- Создание товара (`/sell`): название, описание, цена, категория (Транспорт / Имущество / Вирты / Аксессуары), сервер (1–21)
- Для категории "Вирты" — дополнительные поля: количество (кк) и цена за 1кк
- Редактирование своего объявления
- Удаление объявления (запрещено при активной сделке)
- Валидация всех полей на сервере
- Каталог с фильтрацией по категории, серверу, поиском и сортировкой, пагинацией
- Карточка товара: продавцу показывается "Это ваш товар" вместо кнопки "Купить"

### Сделки
- Создание сделки с карточки товара
- Защита от дублирующихся сделок (если сделка уже есть — редирект на неё)
- Чат между покупателем и продавцом внутри сделки
- Статусы: Ожидание → В работе → Завершено / Спор
- Кнопки по роли и статусу:
  - WAITING: продавец видит "Принять" + "Спор", покупатель — только "Спор"
  - IN_PROGRESS: покупатель видит "Завершить" + "Спор", продавец — только "Спор"
  - DONE / DISPUTE: кнопок нет
- Только участники сделки могут менять её статус

### Баланс
- Актуальный баланс из БД в Header (обновляется при каждой навигации)
- При завершении сделки: атомарное списание у покупателя + зачисление продавцу
- Проверка достаточности средств с понятной ошибкой
- Защита от double-spend через `$transaction` с `Serializable` изоляцией
- Идемпотентность: повторный вызов "Завершить" не списывает дважды

### Споры
- Открытие спора из сделки
- `/disputes` — только свои споры (buyer или seller)
- Бейдж роли: "Вы покупатель" / "Вы продавец"
- "Завершить и оплатить" — покупатель соглашается заплатить
- "Вернуть в работу" — любой участник возвращает сделку в IN_PROGRESS
- Нельзя действовать по чужому спору

### Администрирование
- `/admin/disputes` — доступна только email-адресам из `ADMIN_EMAILS`
- Список всех активных споров из БД с историей чата
- "Покупатель прав" — закрывает сделку без движения денег
- "Продавец прав" — списывает у покупателя, зачисляет продавцу (Serializable транзакция)

### Профиль
- Список своих объявлений с кнопкой удаления
- Баланс загружается из БД (не из JWT)
- `/profile/settings` — смена username (валидация длины, уникальности)

### Избранное
- Добавление/удаление товаров в избранное
- Страница `/favorites` с избранными товарами

## Как запустить проект

### Требования
- Node.js 20+
- PostgreSQL (или Neon / Supabase)

### Установка

```bash
# 1. Клонировать репозиторий
git clone <repo-url>
cd radrp-market

# 2. Установить зависимости
npm install

# 3. Создать .env
cp .env.example .env
# Заполнить DATABASE_URL, NEXTAUTH_SECRET, ADMIN_EMAILS

# 4. Применить схему БД
npx prisma db push

# 5. Запустить в режиме разработки
npm run dev
```

Открыть: http://localhost:3000

### Как стать администратором
Добавить свой email в `.env`:
```
ADMIN_EMAILS="your@email.com"
```
После этого `/admin/disputes` будет доступна.

### Как выдать баланс пользователю (для тестирования)
Пока нет UI — выдать вручную через Prisma Studio:
```bash
npx prisma studio
```
Найти пользователя → изменить поле `balance`.
